Core Components

Designed for scalability, ease of use, and production environments.

my_sqlmodel.py

An enhanced base class extending SQLModel with smart automation and clean data handling.

  • Automatic Pluralization Converts class names (e.g., `UserLog`) to snake_case plural table names (e.g., `user_logs`) automatically using `inflect`.
  • Smart JSON Serialization Custom `to_json()` method that recursively handles nested models, lists, and dicts. No more serialization errors.
  • Date & UUID Handling Built-in support for `datetime` (to ISO format) and `UUID` conversion, making your API responses JSON-safe by default.
  • Pydantic Integration Pre-configured `model_config` with arbitrary types allowed and extra fields ignored.

db.py

A robust database connection manager optimized for both local dev and serverless deployment.

  • Intelligent Pooling Detects `neon.tech` URLs to optimize connection pooling (smaller pool, no timeouts) vs standard PostgreSQL configurations.
  • Zero-Config Localhost Automatically defaults to local credentials when `DB_HOST=localhost`, making setup instant for new developers.
  • Robust Initialization `init_db()` checks for existing tables before creation, handling race conditions and ensuring safe startups without errors.
  • Connection Health Enables `pool_pre_ping` to automatically detect and recover from dropped database connections.
